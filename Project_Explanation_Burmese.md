# NLP Project A5: လုပ်ငန်းစဉ်ဆက်စပ်ရှင်းလင်းချက် (Burmese Explanation)

ဤမှတ်တမ်းသည် NLP ဘာသာရပ်၏ Assignment A5 တွင် အသုံးပြုထားသော နည်းပညာများနှင့် လုပ်ဆောင်ခဲ့သော အဆင့်များကို ရှင်းလင်းတင်ပြထားခြင်းဖြစ်ပါသည်။

## ၁။ ရည်ရွယ်ချက် (Objective)
Large Language Models (LLMs) များသည် တစ်ခါတစ်ရံတွင် မမှန်ကန်သောအချက်အလက်များကို ယုံကြည်လောက်အောင်ဖြေဆိုတတ်သည် (Hallucination)။ ဤလုပ်ငန်းစဉ်သည် ထိုသို့အမှားဖြေဆိုခြင်းများကို လျှော့ချရန်နှင့် မော်ဒယ်အားပိုမိုမှန်ကန်စွာဖြေဆိုနိုင်စေရန် (Alignment) လုပ်ဆောင်ခြင်းဖြစ်သည်။ ထို့အပြင် မော်ဒယ်၏တိုးတက်မှုကို အခြား LLM တစ်ခုကိုအသုံးပြုကာ အကဲဖြတ်ခြင်း (LLM-as-a-Judge) ကိုပါလုပ်ဆောင်ထားပါသည်။

## ၂။ နည်းပညာရပ်များ (Methodology)

### Dataset Preparation (ဒေတာပြင်ဆင်ခြင်း)
ပထမဆုံးအနေဖြင့် `jondurbin/truthy-dpo-v0.11` dataset ကို Hugging Face မှရယူပါသည်။ ၎င်း dataset တွင် Prompt တစ်ခုစီအတွက် မှန်ကန်သောအဖြေ (Chosen) နှင့် မှားယွင်း/ယောင်ယမ်းသောအဖြေ (Rejected) ဟူ၍ နှစ်မျိုးပါဝင်ပါသည်။

### Direct Preference Optimization (DPO)
မော်ဒယ်ကို လေ့ကျင့်သင်ကြားရာတွင် Direct Preference Optimization (DPO) ကိုအသုံးပြုပါသည်။ DPO သည် ရှုပ်ထွေးသော Reinforcement Learning (RLHF) အစား၊ မော်ဒယ်၏ Probability ကိုတိုက်ရိုက်ပြောင်းလဲပေးခြင်းဖြင့် "Chosen" အဖြေများကိုပိုမိုထုတ်ပေးရန်နှင့် "Rejected" အဖြေများကိုရှောင်ကြဉ်ရန် ထိရောက်လွယ်ကူစွာ သင်ကြားပေးသောနည်းလမ်းဖြစ်ပါသည်။

ထိုသို့လေ့ကျင့်ရန်အတွက် `Qwen/Qwen2.5-1.5B-Instruct` မော်ဒယ်ကိုအသုံးပြုထားပြီး၊ Memory သုံးစွဲမှုသက်သာစေရန်အတွက် LoRA (Low-Rank Adaptation) နည်းပညာဖြင့် PEFT (Parameter-Efficient Fine-Tuning) ကို တွဲဖက်အသုံးပြုထားပါသည်။ 

### LLM-as-a-Judge ဖြင့်အကဲဖြတ်ခြင်း (Evaluation)
မော်ဒယ်ကိုလေ့ကျင့်ပြီးနောက်၊ ၎င်း၏အရည်အသွေးကို AlpacaEval benchmark ဖြင့်စမ်းသပ်ပါသည်။ တူညီသော Prompt များကို (အသစ်လေ့ကျင့်ထားသော မော်ဒယ် - DPO Model) နှင့် (မူလမော်ဒယ် - Base Model) နှစ်ခုလုံးအား ဖြေဆိုစေပါသည်။
ထို့နောက် ထွက်လာသောအဖြေနှစ်ခုကို အကဲဖြတ်သူ (Judge) အဖြစ်တာဝန်ယူသော ပိုမိုစွမ်းအားကြီးသည့် LLM (ဥပမာ - GPT-4o-mini သို့မဟုတ် Gemini) သို့ပေးပို့ကာ၊ မည်သည့်အဖြေကပိုမိုကောင်းမွန်သနည်းဆိုသည်ကို ဆုံးဖြတ်ခိုင်းပါသည်။ ထိုရလဒ်များကို Win Rate အဖြစ်ရာခိုင်နှုန်းဖြင့် တွက်ချက်တင်ပြထားပါသည်။

## ၃။ အခြားလုပ်ဆောင်ချက်များ (Additions)
* **Web App (Progress Flow)**: လုပ်ငန်းစဉ်အဆင့်ဆင့်ကို ရှင်းလင်းစွာမြင်သာစေရန် HTML, CSS, JS တို့ဖြင့် Interactive Web Application တစ်ခုကို ရေးသားဖန်တီးထားပါသည်။
* **Python Notebook**: လိုအပ်သောလုပ်ဆောင်ချက်အားလုံး (Training, Pushing, Evaluation) ကို Notebook တစ်ခုတည်းတွင် အလွယ်တကူ Run နိုင်ရန် စနစ်တကျရေးသားထားပါသည်။ Environment အပေါ်မူတည်ပြီး GPU (CUDA, MPS, CPU) ကို အလိုအလျောက်ရွေးချယ်ပေးနိုင်ပါမည်။
