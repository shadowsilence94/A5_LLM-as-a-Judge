Project A5 Progress Flow

Optimization of Human Preference and Evaluating Factuality by LLM-as-a-Judge

Task 1: Dataset Preparation

Loading the core truth/hallucination classification dataset.

Action: Utilizing jondurbin/truthy-dpo-v0.1 from the Hugging Face hub.

Purpose: To map prompts to "chosen" (factual) and "rejected" (hallucinated) responses, providing a clear pairwise optimization signal.

Task 2: Model Training via DPO

Direct Preference Optimization using Hugging Face TRL and LoRA Adapters.

Action: Implementing DPOTrainer on the Qwen/Qwen2.5-1.5B-Instruct architecture.

Purpose: Adjusts model likelihoods intrinsically, avoiding complex Reinforcement Learning with Human Feedback (RLHF) loops while still perfectly aligning the model to factual responses.

Task 3: Hugging Face Hub Integration

Persisting checkpoints and sharing the verified weights.

Action: Employing the native huggingface_hub Python API with local tokens natively.

Purpose: Allows immediate access to the fine-tuned model weights across varying runtime environments seamlessly.

Task 4: Evaluation: LLM-as-a-Judge

Automated evaluation protocol leveraging AlpacaEval metrics.

Action: Running structured inference against a deterministic Judge LLM (such as GPT-4o-mini).

Purpose: Compares the baseline model versus the newly trained DPO variant, aggregating pairwise judgments to compute a final, quantifiable Win Rate.
